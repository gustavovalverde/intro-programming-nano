The HTML Language

The Markup part of HTML is because this is a markup language made of markup tags. Each of the HTML tags describes a different document content.

Here's a basic example:

  1. <!DOCTYPE html>
  2. <html>
  3. <head>
  4. <title>Page Title</title>
  5. </head>
  6. <body>
  7. <h1>My First Heading</h1>
  8. <p>My first paragraph.</p>
  9. </body>
  10. </html>
  • Tag: An HTML tag is always contained within angled brackets. Most tags have an opening tag and a closing tag. Some tags (called "void" tags) do not require a closing tag.
  • Element: An HTML element refers to everything within a set of opening and closing tags.
  • Attribute: This is a property of an HTML element. For example, to set the href attribute of an anchor tag to the Udacity URL, you would write <a href="www.udacity.com">Udacity Web Page<a>

If you'd like to know more about what you can do with each HTML tag, you can always find the best references in W3 Schools, try it!

From the HTML "tree" to the "DOM"

When you start building your webpage, you might realized that the elements of a page are arranged in a tree-like hierarchical structure, where each element has a parent and child elements. Although we call it a tree-like structure, in a tree the root [element] is at the bottom while the "last" child (or childs) is right at the top, as opposed to HTML where the rootelement is at the top of the tree.You can use the developer tools in Chrome to navigate through the tree structure. Press ctrl-shift-I to start it, then look to the top right of the screen to find the elements of your page. If you select an element it will be highlighted onscreen and its properties displayed. This might be the most useful tool for understanding and editing HTML and CSS.

Now, going deeper into the HTML tree-like structure, there's a more important, but also confusing, concept in the Web world which is the DOM (The Document Object Model), this concept is difficult to understand not just for beginners but also for advance developers, to whom the concept of DOM is like a black box. You can read the techy definition here, but that might be more confusing than helpful if you don't understand other concepts like an API.

The first problem when people try to figure out the difference between the DOM and their actual HTML code is that both are almost the same. The critical difference between the ideas of DOM and your HTML code is that the DOM can only exist at run time, inside the browser. The browser builds an understanding of your code when it reads it, that understanding is called the DOM and it exists inside the browser. The tags in HTML represent instructions to the browser about how to build the DOM and what different parts of the DOM are there.

Making HTML "Pretty" with CSS

CSS stands for Cascading Style Sheets and this is the code written to control the style of your HTML elements. This is done by giving similar HTML elements the same class name and then specifying the style that should apply to that class, for example. Inheretance is a key feature in CSS, and by using it web developers can achieve two things:

  • Avoid errors: CSS helps by avoiding repetition, this way web developers don't have to do the same things over and over to style their site. This reduces the probability of errors when the developer decides to make a change to something.
  • Consistency: Instead of manually giving each one the HTML tags the same style, web developers are able to assign different div's the same class and then specify the style for that class in only one place.
